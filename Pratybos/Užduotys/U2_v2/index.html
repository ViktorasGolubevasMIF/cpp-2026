    U2: OOP Pagrindai. Klasės ir Objektai

    Savaitės: 3-4
    Svoris: 1 balas

    Versija: v.2

    👉 Užduočių Atlikimo ir Pateikimo Gidas

    🎯 Užduoties tikslas

    Išmokti kurti C++ klases, jų egzempliorius - objektus; suvokti inkapsuliacijos principą; suprasti aprašų ir apibrėžimų atskyrimą klasės (loginiame) ir modulio (fiziniame) lygmenyse; dirbti su konstruktoriais ir destruktoriais; panaudoti statinius ir masyvinius klasių (objektų) narius.

    📚 Mokymosi tikslai

    Atlikę šią užduotį, mokėsite:

    • ✅ Transformuoti C struktūrą į C++ klasę (struct į class)
    • ✅ Naudoti private ir public prieigos modifikatorius
    • ✅ Atskirti aprašus nuo apibrėžimų klasės (loginiame) ir modulio (fiziniame) lygmenyse
    • ✅ Kurti konstruktorius (default ir parametrinius)
    • ✅ Kurti ir naudoti getterius/setterius
    • ✅ Naudoti statinius narius (klasės lygio duomenys)
    • ✅ Rašyti destruktorius su logging
    • ✅ Dirbti su C-style masyvais klasėje
    • ✅ (Bonus) pereiti nuo masyvo prie vector

    📐 Užduoties formatas

    Žingsnių struktūra:

    • 5 pagrindiniai žingsniai + 1 papildomas žingsnis
    • Žingsniai 1-2: Pilni pavyzdžiai (apšilimas)
    • Žingsniai 3-6: Jūsų kodas pagal užduotis

    Kodo žymėjimai:

    • PAVYZDYS — pilnas veikiantis kodas (galite kopijuoti)
    • JŪSŲ KODAS — turite patys sukurti
    • // UŽDUOTIS: — komentarai nurodo, ką reikia padaryti

    Pavyzdys: 

    void pridetiPazymi(double p) {
    // UŽDUOTIS: patikrinti ar p tinkamas (0.0 - 10.0)
    // UŽDUOTIS: pridėti į masyvą
}

    Tai reiškia — jūs rašote kodą, kuris atlieka tai, kas aprašyta komentaruose.

    Direktorija: U2/01/

    Reikalavimai:

    • ✅ Sukurkite 2 failus tame pačiame žingsnyje, kad būtų matoma transformacija:

    Failas student_struct.cpp - C struktūros versija:

    #include <iostream>
#include <cstring>
using namespace std;

struct Student {
    char vardas[50];
    int amzius;
    double pazymys;

    Failas student_class.cpp - C++ klasės versija (su public nariais):

    #include <iostream>
#include <cstring>
using namespace std;

class Student {
public:  // Visi nariai public (kaip struct)
    char vardas[50];

    • class - visi nariai private pagal nutylėjimą
    • Parodyti skirtumą tarp struct (viską galima pasiekti) ir class (reikia metodų)

    2 žingsnis: Konstruktoriai (pateiktas detalesnis sprendimas negu pradiniame variante)

    Direktorija: U2/02/

    Reikalavimai:

    • ✅ Sukurkite Student klasę su konstruktoriais.
    • ✅ Klasę Student "patalpinkite" į modulį Student (.h ir .cpp failus)

    A. Pradžia: Viskas main.cpp (inline konstruktoriai)

    Pirmiausia sukurkite klasę viename faile su konstruktoriais viduje klasės:

    main.cpp:

    #include <iostream>
#include <cstring>
using namespace std;

class Student {
private:
    char vardas[50];
    int amzius;
    double pazymys;

public:
    // Konstruktoriai VIDUJE klasės (inline)
    Student() {
        strcpy(vardas, "Nezinomas");
        amzius = 0;
        pazymys = 0.0;
        cout << "[TEST] Student sukurtas (default): " << vardas << endl;
    }

    Student(const char* v, int a, double p) {
        strcpy(vardas, v);
        amzius = a;
        pazymys = p;
        cout << "[TEST] Student sukurtas: " << vardas << endl;
    }
};

int main() {
    Student s1;  // Default konstruktorius
    Student s2("Jonas", 20, 8.5);  // Parametrinis
    Student s3("Petras", 21, 9.0);

    return 0;
}

    Kompiliavimas: 

    g++ -Wall -std=c++11 main.cpp -o programa
./programa

    Išvestis: 

    [TEST] Student sukurtas (default): Nezinomas
[TEST] Student sukurtas: Jonas
[TEST] Student sukurtas: Petras

    ✅ Veikia! Bet yra problema...

    Problema: Aprašai ir apibrėžimai kartu

    Inline konstruktoriai (viduje klasės) "sumaišo" aprašus ir apibrėžimus vienoje vietoje.

    C/C++ principas:

    • Aprašas (funkcijos prototipas) — KAS tai (interface)
    • Apibrėžimas (funkcijos "kūnas") — KAIP tai veikia (implementation)
    • Jie turėtų būti atskirti

    Sprendimas: Iškelti apibrėžimus už klasės ribų.

    B. Bandymas: Iškelti už klasės BE ::

    main.cpp (BANDYMAS - su klaida!):

    #include <iostream>
#include <cstring>
using namespace std;

class Student {
private:
    char vardas[50];
    int amzius;
    double pazymys;

public:
    // TIK APRAŠAI (be apibrėžimų)
    Student();
    Student(const char* v, int a, double p);
};

// BANDOME apibrėžti už klasės BE :: - NEVEIKS!
Student() {  // KLAIDA!
    strcpy(vardas, "Nezinomas");
    amzius = 0;
    pazymys = 0.0;
    cout << "[TEST] Student sukurtas (default): " << vardas << endl;
}

Student(const char* v, int a, double p) {  // KLAIDA!
    strcpy(vardas, v);
    amzius = a;
    pazymys = p;
    cout << "[TEST] Student sukurtas: " << vardas << endl;
}

int main() {
    Student s1;
    Student s2("Jonas", 20, 8.5);
    return 0;
}

    Kompiliavimas: 

    g++ -Wall -std=c++11 main.cpp -o programa

    Kompiliavimo klaida: 

    main.cpp: In function 'int Student()':
main.cpp:19:12: error: 'vardas' was not declared in this scope
   19 |     strcpy(vardas, "Nezinomas");
      |            ^~~~~~
main.cpp:20:5: error: 'amzius' was not declared in this scope
   20 |     amzius = 0;
      |     ^~~~~~
main.cpp:21:5: error: 'pazymys' was not declared in this scope
   21 |     pazymys = 0.0;
      |     ^~~~~~~

    Kodėl? Kompiliatorius mano, kad Student() yra globali funkcija, kuri grąžina int tipą. Be :: jis nemato ryšio su klase ir negali pasiekti private narių!

    C. Sprendimas: Scope resolution operatorius ::

    main.cpp (TEISINGAI su ::):

    #include <iostream>
#include <cstring>
using namespace std;

class Student {
private:
    char vardas[50];
    int amzius;
    double pazymys;

public:
    // TIK APRAŠAI
    Student();
    Student(const char* v, int a, double p);
};

// :: nurodo KLASĖS kontekstą - VEIKIA!
Student::Student() {
    strcpy(vardas, "Nezinomas");
    amzius = 0;
    pazymys = 0.0;
    cout << "[TEST] Student sukurtas (default): " << vardas << endl;
}

Student::Student(const char* v, int a, double p) {
    strcpy(vardas, v);
    amzius = a;
    pazymys = p;
    cout << "[TEST] Student sukurtas: " << vardas << endl;
}

int main() {
    Student s1;
    Student s2("Jonas", 20, 8.5);
    Student s3("Petras", 21, 9.0);
    return 0;
}

    Kompiliavimas: 

    g++ -Wall -std=c++11 main.cpp -o programa
./programa

    ✅ Dabar veikia!

    Konteksto :: operatorius:

    • Student::Student() = "konstruktorius, kuris priklauso Student klasei"
    • :: = scope resolution operatorius (konteksto nurodymas)
    • Be :: — globali funkcija
    • Su :: — klasės narys

    ✅ Dabar turime klasės lygmens (loginį) atskyrimą — aprašai (klasės viduje) vs apibrėžimai (už klasės su ::)

    D. Fizinis atskyrimas į .h ir .cpp

    Po loginio atskyrimo, galime atlikti fizinį atskyrimą į modulinę struktūrą:

    Student.h (aprašai):

    #ifndef STUDENT_H
#define STUDENT_H

class Student {
private:
    char vardas[50];
    int amzius;
    double pazymys;

public:
    // TIK APRAŠAI
    Student();
    Student(const char* v, int a, double p);
};

#endif

    Student.cpp (apibrėžimai):

    #include "Student.h"
#include <iostream>
#include <cstring>

// Apibrėžimai su ::
Student::Student() {
    strcpy(vardas, "Nezinomas");
    amzius = 0;
    pazymys = 0.0;
    std::cout << "[TEST] Student sukurtas (default): " << vardas << std::endl;
}

Student::Student(const char* v, int a, double p) {
    strcpy(vardas, v);
    amzius = a;
    pazymys = p;
    std::cout << "[TEST] Student sukurtas: " << vardas << std::endl;
}

    main.cpp:

    #include <iostream>
#include "Student.h"
using namespace std;

int main() {
    Student s1;
    Student s2("Jonas", 20, 8.5);
    Student s3("Petras", 21, 9.0);
    return 0;
}

    Kompiliavimas: 

    g++ -Wall -std=c++11 -c main.cpp
g++ -Wall -std=c++11 -c Student.cpp
g++ main.o Student.o -o programa
./programa

    ✅ Galutinis rezultatas — klasė modulyje!

    Dviejų lygmenų atskyrimas:

    • Loginis lygmuo (klasės kontekstas):
      • Inline (viduje klasės): metodai apibrėžti klasės kūne — :: nereikia
      • Už klasės: aprašai klasėje, apibrėžimai už jos — reikia ::
    • Fizinis lygmuo (modulinis):
      • .h failas — tik aprašai (interface, KAS tai)
      • .cpp failas — tik apibrėžimai (implementation, KAIP tai veikia)

    Scope resolution :::

    • Student::Student() — konstruktorius priklauso Student klasei
    • Be :: kompiliatorius mano, kad tai globali funkcija → klaida
    • Su :: — klasės narys (teisingai!)

    Išvestis: 

    [TEST] Student sukurtas (default): Nezinomas
[TEST] Student sukurtas: Jonas
[TEST] Student sukurtas: Petras

    Direktorija: U2/04/

    return sukurtaStudentu;
}

    Technikos: - Static narys - bendras visiems objektams (objektai neturi šio nario (šios dalies) savyje) - Inicializuoti už klasės ribų: int Student::sukurtaStudentu = 0; - Destruktorius - automatiškai kviečiamas, kai objektas "miršta" (baigiasi jo gyvavimo ciklas) - Alternatyva: Inicializavimo sąrašas konstruktoriuje (modernesnė C++ praktika): 

    Student::Student(const char* v, int a, double p) 
    : amzius(a), pazymys(p)  // Initialization list
{
    strcpy(vardas, v);  // char[] reikia body'je
    sukurtaStudentu++;
}

    Testas: 

    int main() {
    cout << "Studentų: " << Student::gautiSukurtaStudentu() << endl;  // 0

    {
        Student s1("Jonas", 20, 8.5);
        Student s2("Petras", 21, 9.0);
        cout << "Studentų: " << Student::gautiSukurtaStudentu() << endl;  // 2
    }  // s1 ir s2 sunaikinami čia

    cout << "Studentų: " << Student::gautiSukurtaStudentu() << endl;  // 0

    return 0;
}

    💡Svarbu!

    • Naudojame static const int MAX_PAZYMIU = 20;, o ne #define MAX_PAZYMIU 20
    • Tai klasės konteksto konstanta (ne objekto!), kaip ir kintamieji static nariai (viena visiems objektams)
    • Skirtumas nuo sukurtaStudentu:
    • static const int — inicializuojama inline (klasėje)
    • static int — inicializuojama out-of-line (už klasės: int Student::sukurtaStudentu = 0;)
    • Kodėl const gali inline? Kompiliatorius žino, kad reikšmė nekis
    • Plačiau apie konstantiškumą — vėlesnėse temose

    ❓ Dažnai užduodami klausimai

    K: Kodėl strcpy() vietoj string?
    A: Demonstruoti C-style string'us (tradicinė sintaksė). Vėliau (U3+) naudosime std::string.

    K: Ar privaloma naudoti const metodams?
    A: Taip, getters ir utility metodai turi būti const (gera praktika).

    K: Kodėl static narys inicializuojamas už klasės?
    A: Nes static narys - bendras visiems objektams, ne kiekvieno objekto dalis. Reikia vienos kopijos visai klasei.

    K: Ar destruktorius visada automatinis?
    A: Taip, destruktorius automatiškai kviečiamas, kai objektas išeina iš scope (arba delete jei dinaminis).

    K: Ar žingsnis 6 (bonus) privalomas?
    A: Ne, tai bonus (+0.2 balo). Bet rekomenduojamas visiems studentams.
