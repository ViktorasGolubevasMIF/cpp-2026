    U3: Kompozicija (has-a ryšys)

    Savaitės: 5-6
    Svoris: 1 balas
    Terminas: Savaitės 6 pabaiga

    👉 Žr. Užduočių Gidas

    🎯 Užduoties tikslas

    Išmokti kurti sudėtingesnes klases su kompozicija (has-a ryšys).

    📚 Mokymosi tikslai

    Atlikę šią užduotį, mokėsite:

    • ✅ Kurti klases, kurios "turi"/talpina kitas klases (kompozicija)
    • ✅ Dirbti su rodyklėmis į objektus (Objektas*)
    • ✅ Dirbti su vector<Objektas> (objektų kolekcija)
    • ✅ Naudoti statinius narius (klasės lygio duomenys)
    • ✅ Kurti globalias funkcijas (ne metodus)
    • ✅ Suprasti kompozicijos vs agregacijos skirtumus

    💡 Kas yra kompozicija?

    Kompozicija - tai struktūrinis ryšys/santykis, kai vienas objektas turi/patalpina savyje (contains) kitą objektą.

    Pavyzdys:

    • Kambarys HAS Langą (has-a)
    • Kambarys HAS Duris (has-a)

    Skirtumas nuo paveldėjimo:

    • Paveldėjimas: Automobilis IS-A TransportoPriemone (vėlesnė tema)
    • Kompozicija: Automobilis HAS-A Variklis (ši tema)

    Šios užduoties kompozicinė objektų struktūra: 

    Kambarys (konteineris/talpykla)
  ├─ Langai (komponentai - naudojama REIKŠMĖS semantika/prasmė)
  │   ├─ Langas 1
  │   ├─ Langas 2
  │   └─ ...
  └─ Durys (komponentai - naudojama RODYKLĖS semantika)
      └─ Durys*

    Room (container)
  ├─ Windows (components - value semantics)
  │   ├─ Window 1
  │   ├─ Window 2
  │   └─ ...
  └─ Doors (component - pointer semantics)
      └─ Door*

    📋 Užduoties žingsniai

    1 žingsnis: Langas ir Durys klasės

    Direktorija: U3/01/

    Reikalavimai:

    • ✅ Sukurkite 2 paprastas klases (panaudokite U2 įgūdžius)

    Klasė Langas:

    private atributai/savybės (duomenys-nariai):

    • float aukstis - lango aukštis (metrais)
    • float plotis - lango plotis (metrais)
    • static int sukurtaLangu - statinis skaitiklis (kiek langų sukurta)

    public metodai (funkcijos-nariai):

    • Langas(float a, float p) - konstruktorius (parametrinis)
      • Inicializuoja aukstis, plotis
      • Padidina sukurtaLangu++
      • Logging: [DEBUG] Langas sukurtas...
    • ~Langas() - destruktorius
      • Sumažina sukurtaLangu--
      • Logging: [DEBUG] Langas sunaikintas...
    • float gautiAuksti() const - getter'is
    • float gautiPloti() const - getter'is
    • float apskaiciuotiPlota() const - aukštis × plotis
    • static int gautiSukurtaLangu() - klasės lygio getter (statinis)

    Klasė Durys:

    Analogiškai kaip Langas:

    • private: aukstis, plotis, static int sukurtaDuru
    • public: konstruktorius, destruktorius, atributų getter'iai, apskaiciuotiPlota(), statinis skaitliuko getter'is

    Svarbu!

    Kodėl nėra default konstruktoriaus?

    • Langas ir Durys visuomet turi turėti matmenis
    • Default konstruktorius (Langas()) sukurtų "nelogiškus" objektus (0×0)
    • Geroji praktika: jei objektui būtina pradinė būsena — parametrinis konstruktorius

    Bandymas: 

    Langas langas;  // ❌ KLAIDA: no matching constructor

    Tai sąmoningas klasės suprojektavimas — langai be matmenų neegzistuoja!

    Testavimas:

    2 žingsnis: Refaktoringas į tarptautines praktikas

    Direktorija: U3/02/

    Motyvacija:

    Lietuviški pavadinimai geri mokantis, bet realiuose projektuose:

    • Anglų kalba — standartas
    • Skirtingos komandos naudoja skirtingas vardų konvencijas
    • Reikia susipažinti su abiem praktikomis

    Reikalavimai:

    Pakeiskite Langas ir Durys klases panaudojant:

    • Angliškus failų, klasių vardus (Window, Door)
    • Angliškus duomenų ir metodų (bent jau getter/setter) pavadinimus
    • Skirtingas narių užvadinimo (member naming) praktikas

    A. Window klasė — this-> praktika

    Window.h: 

    #ifndef WINDOW_H
#define WINDOW_H

class Window {
private:
    float height;
    float width;
    static int windowCount;

public:
    Window(float h, float w);
    ~Window();

    // UŽDUOTIS: Sukurti getters
    float getHeight() const;
    float getWidth() const;
    float calculateArea() const;

    static int getWindowCount();

#endif

    Window.cpp — PAVYZDYS su this->: 

    #include "Window.h"
#include <iostream>

int Window::windowCount = 0;

// PAVYZDYS: konstruktorius su this->
Window::Window(float height, float width) {
    // Parametrai turi TĄ PATĮ vardą kaip member'iai
    this->height = height;  // this-> nurodo į member
    this->width = width;

    windowCount++;
    std::cout << "[DEBUG] Window created (" << this->height 
              << "×" << this->width << "). Total: " 
              << windowCount << std::endl;
}

Window::~Window() {
    windowCount--;
    std::cout << "[DEBUG] Window destroyed. Remaining: " 
              << windowCount << std::endl;
}

// JŪSŲ KODAS - getters:
float Window::getHeight() const {
    // UŽDUOTIS: grąžinti height
}

float Window::getWidth() const {
    // UŽDUOTIS: grąžinti width
}

float Window::calculateArea() const {
    // UŽDUOTIS: grąžinti height * width
}

int Window::getWindowCount() {
    return windowCount;
}

    Technikos:

    Naming conventions:

    • getX() / setX() — Java/C# stilius (populiariausias)
    • get_x() / set_x() — Python/snake_case (rečiau C++)
    • Lietuviški (gautiX()) — mokymuisi, ne produkcijai

    Member vs parametrų vardai:

    1. this-> naudojimas (Window):

      • Parametras ir member'is turi tą patį vardą
      • this->height = height; — aiškiai atskiria
      • Privalumas: natūralūs vardai

    2. Underscore prefix (Door):

      • Member'iai: _height, parametrai: h arba height
      • Nereikia this->
      • Privalumas: aiškus skirtumas kode

    3. Skirtingi vardai:

      • Member: height, parametras: h
      • Paprasčiausias, bet ne visada aiškus

    🆕 NAUJAS OPERATORIUS — new / delete:

    • new Type — sukuria objektą dinamiškai (heap'e)
    • delete pointer — atlaisvina atmintį (BŪTINA!)
    • Skirtumas nuo stack:
      • Stack: Door door(2.0, 0.9); — automatiškai sunaikinama
      • Heap: Door* p = new Door(2.0, 0.9); — reikia delete!
    • Memory leak — jei pamiršite delete

    nullptr (C++11+):

    • ✅ nullptr — type-safe, modernas
    • ⚠️ NULL — C makro (legacy)
    • ❌ 0 — integer, ne pointer

    Inkapsuliacijos principas = Informacijos slėpimas (dažna interpretacija):

    • Private nariai nepasiekiami iš išorės
    • Privaloma naudoti public interfeisą (getters/setters)
    • Bandymas window.height → kompiliavimo klaida
    • Tai apsaugo duomenis ir leidžia validaciją

    Inkapsuliacijos ir informacijos slėpimo skirtumas (tradicinė interpretacija):

    • Inkapsuliacijos (encapsulation) — duomenų ir metodų sujungimas į vieną vienetą (klasę)
    • Informacijos slėpimas (information hiding) — private/public mechanizmas

    3 žingsnis: Room klasė (kompozicija)

    Direktorija: U3/03/

    Motyvacija:

    Iki šiol turėjome paprastas klases (Window, Door).

    Dabar kursime sudėtingesnę klasę, kuri naudoja du būdus "turėti"/"susidėti" informaciją/duomenis apie objektus: 1. patalpindami (klasės išorėje sukurtų) objektų kopijas 2. saugodami rodykles į (klasės išorėje sukurtus) objektus

    Realaus pasaulio analogija: Kambarys "turi" langus ir duris — tai natūrali has-a struktūra!

    Svarbu!

    Prieš pradedant šį žingsnį, užkomentuokite Window ir Door klasių konstruktorių/destruktorių [DEBUG] logging išvestis: 

    Window::Window(float height, float width) {
    this->height = height;
    this->width = width;
    windowCount++;
    // std::cout << "[DEBUG] Window created..." << std::endl;  ← Užkomentuota
}

Window::~Window() {
    windowCount--;
    // std::cout << "[DEBUG] Window destroyed..." << std::endl;  ← Užkomentuota
}

    Kodėl?

    • Dabar susikoncentruojame į Room kompoziciją
    • "Virtuvė" (objektų kopijavimas, copy semantics) bus atskleista vėliau (bonus žingsnyje)
    • Naudojame natūralius C++ primityvus kompoziciją pademonstruoti be [DEBUG] "triukšmo"

    Reikalavimai:

    Sukurkite Room klasę, kuri susideda iš:

    Privatūs nariai:

    • float length — kambario ilgis (m)
    • float width — kambario plotis (m)
    • float height — kambario aukštis (m)
    • vector<Window> windows — langų kolekcija (objektų kopijos)
    • Door* door — rodyklė į duris (saugome tik adresą)

    Vieši metodai:

    Konstruktorius:

    • Room(float l, float w, float h)
    • Inicializuoja matmenis (atributus)
    • Inicializuoja door = nullptr (kol kas durų nėra)
    • Logging: [DEBUG] Room created...

    Destruktorius:

    • ~Room()
    • Logging: [DEBUG] Room destroyed
    • Pastaba: vector automatiškai sunaikina Window objektus
    • Door nenaikinome — ji išorėje (main.cpp atsakingas)

    Kompozicijos metodai:

    • void addWindow(const Window& w)

      • Prideda langą į windows kolekciją
      • Kas nutinka? vector sukuria Window kopiją ir laiko ją savyje
      • Kodėl const Window&? Efektyvumas — objektą perduodame pagal nuoodą, ne pagal reikšmę (SVARBU!)
      • Logging: [INFO] Window added. Total: X

    • void setDoor(Door* d)

      • Nustato durų rodyklę
      • Kas nutinka? Room laiko tik adresą, ne patį objektą
      • Door objektas yra sukurtas su new išorėje (main.cpp)

    Getter'iai:

    • float getLength() const — kambario ilgis
    • float getWidth() const — kambario plotis
    • float getHeight() const — kambario aukštis
    • int getNumberOfWindows() const — langų skaičius (windows.size())
    • const Door* getDoor() const — durų rodyklė (read-only)

         << "×" << room.getWidth()

    cout << "Windows: " << room.getNumberOfWindows() << endl;

    if (room.getDoor() != nullptr) {
        cout << "Door: " << room.getDoor()->getHeight() 
             << "×" << room.getDoor()->getWidth() << " m" << endl;
    }

    cout << "\n--- Atlaisviname atmintį ---" << endl;
    delete door;  // main.cpp atsakingas!

    cout << "\n--- Programa baigiasi (Room sunaikinamas) ---" << endl;
    return 0;
}

    Technikos:

    Du būdai saugoti objektus:

    1. Kopija (vector<Window> windows):

      • addWindow(w1) — nukopijuoja visą Window objektą
      • Room laiko Window kopijas savyje
      • Destruktoriuje vector automatiškai sunaikina langus
      • Analogija: Kambarys turi savo langus

    2. Rodyklė (Door* door):

      • setDoor(d) — laiko tik adresą
      • Door objektas yra išorėje (main.cpp sukūrė su new)
      • Room destruktorius NENAIKINA durų
      • main.cpp atsakingas už delete door
      • Analogija: Kambarys žino, kur yra durys, bet jos ne jo nuosavybė

    Kompozicijos atsakomybės (DUK):

    ❓ Ar Room turi duris?

    • Ne! Room tik žino apie duris (laiko rodyklę)
    • Durys priklauso main.cpp (kas sukūrė su new)

    ❓ Kas atlaisvina door atmintį?

    • main.cpp daro delete door (kas sukūrė, tas ir naikina)
    • Room destruktorius jų neliečia

    ❓ Kas atsitinka su windows?

    • vector automatiškai sunaikina visus Window objektus
    • Room destruktoriuje nieko papildomo daryti nereikia

    Kompozicija vs Agregacija:

    Šiame žingsnyje matėme ABU:

    Kompozicija (vector<Window> windows):

    • Room valdo Window gyvavimo ciklą
    • Window negali egzistuoti be Room
    • Room sunaikinus → automatiškai sunaikinami ir Window
    • "has-a" (strong ownership)

    Agregacija (Door* door):

    • Room žino apie Door, bet nevaldo
    • Door gali egzistuoti nepriklausomai
    • Room sunaikinus → Door lieka (main.cpp valdo)
    • "knows-about" (weak relationship)

    Kada ką naudoti?

    • Kompozicija: komponentas priklauso tik šiam konteineriui
    • Agregacija: komponentas dalinamas ar egzistuoja nepriklausomai

    Pavyzdys:

    • Automobilis turi variklį (kompozicija) — variklis nepriklauso be auto
    • Automobilis turi vairuotoją (agregacija) — vairuotojas gali egzistuoti atskirai

    4 žingsnis: Globalios funkcijos (skaičiavimai)

    Direktorija: U3/04/

    Motyvacija:

    Kambariui reikia skaičiavimų (sienų plotas, grindjuosčių ilgis). Ar šie metodai turėtų būti Room klasėje?

    Ne! Tai atskirti funkcionalumai → globalios funkcijos (ne metodai).

    Kodėl globalios?

    • Separation of concerns — Room neturi "žinoti" apie statybos skaičiavimus
    • Galima skirtingi algoritmai (pvz., su/be angokraščių)
    • Testabilumas — lengviau testuoti atskirai

    Reikalavimai:

    Sukurkite papildomą modulį calculations.h + calculations.cpp su dviem funkcijomis:

    1. Sienų ploto skaičiavimas: 

    float calculateWallArea(const Room& room);

    Algoritmas:

    • Perimetras = 2 × (ilgis + plotis)
    • Sienų plotas = perimetras × aukštis
    • Atimti visų langų plotus
    • Atimti durų plotą

    2. Grindjuosčių ilgio skaičiavimas: 

    float calculateBaseboardLength(const Room& room);

    Algoritmas:

    • Perimetras = 2 × (ilgis + plotis)
    • Atimti durų plotį

    Technikos:

    Globalios funkcijos vs metodai:

    • Metodas — priklauso klasei (room.calculate())
    • Globali — nepriskiriama klasei (calculate(room))
    • Kada naudoti globalias? Kai funkcionalumas ne klasės atsakomybė

    const reference parametras:

    • const Room& room — efektyvumas (nekopijuoja objekto)
    • const — garantuoja, kad funkcija nekeičia objekto

    Inkapsuliacijos principas:

    • Globalios funkcijos negali pasiekti private
    • Privaloma naudoti public interfeisą (getters)
    • Jei reikia daugiau duomenų → pridėti getter

    Separation of concerns:

    • Room saugo duomenis
    • Globalios funkcijos skaičiuoja pagal tuos duomenis
    • Kiekvienas turi savo atsakomybę
