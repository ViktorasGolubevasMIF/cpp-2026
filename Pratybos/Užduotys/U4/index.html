    U4: Trijų Taisyklė ir Operatorių Perkrovimas

    Savaitės: 7-8(-9)
    Svoris: 1 balas (+ BONUS: 0,2 ?)
    Terminas: 2026 balandžio 1 (pratęsta iki: 2026 balandžio 15)

    🎯 Užduoties tikslas

    Išmokti taikyti principą Trijų Taisyklė (Rule of Three) klasėms, kurios valdo dinaminę atmintį.

    📚 Mokymosi tikslai

    Atlikę šią užduotį, mokėsite:

    • ✅ Suprasti paviršinio (shallow) vs giliojo (deep) Kopijos skirtumus
    • ✅ Implementuoti Kopijos konstruktorių (copy constructor) su dinamine atmintimi
    • ✅ Implementuoti Kopijos priskyrimo operatorių (copy assignment operator)
    • ✅ Taikyti Trijų Taisyklės (Rule of Three) principą
    • ✅ Dirbti su C stiliaus eilutėmis (C-style strings) char* C++ priemonėmis
    • ✅ Perkrauti operatorius (operator overloading)

    📝 Pastaba apie komentarus kode

    Šioje užduotyje naudosime // TODO: komentarus, kurie žymi užduotis, kurias jums reikia atlikti.

    TODO (angl. "to do" — "padaryti") yra tarptautinė programavimo praktika — tokius komentarus rasite bet kuriame profesionaliame projekte. IDE (pvz., VS Code, CLion) net specialiai paryškina TODO komentarus, kad lengviau juos pastebėtumėme.

    💡 Kas yra Trijų Taisyklė (Rule of Three)?

    U3 užduotyje susidūrėme su Kopijos konstruktoriaus (copy constructor) problema naudodami vector<Window>. Matėme, kad objektai kopijuojami automatiškai, ir reikėjo vartotojo apibrėžto (user-defined) Kopijos konstruktoriaus (kad pasiaiškinti programos elseną, jos "virtuvę").

    U4 užduotyje grįžtame prie pagrindų — "pamiršime" vector, Room, kompoziciją. Naudosime paprastas klases su dinamine atmintimi (IntArray ir MyString), kad suprastume, kaip teisingai kopijuoti objektus su dinamine atmintimi (t.y. objektus tokių klasių, kurios numato dinaminės atminties panaudojimą).

    Trijų Taisyklė (Rule of Three) — C++ taisyklė: jei klasei reikia vieno iš šių trijų, reikia visų trijų:

    1. Destruktorius (destructor: ~ClassName())
    2. Kopijos konstruktorius (copy constructor: ClassName(const ClassName&))
    3. Kopijos priskyrimo operatorius (copy assignment operator: operator=)

    Kodėl? Kai klasė valdo dinaminę atmintį, numatytasis (default) kopijavimas įtakoja dvigubą (atminties) atlaisvinimą (double delete) ir (paprastai) sukelia programos griūtį/lūžį (crash).

    📋 Užduoties struktūra

    DALIS 1: IntArray (žingsniai 1-3) - Problema → Kopijos konstruktorius → Trijų Taisyklė

    DALIS 2: MyString (žingsniai 4-6) - Trijų Taisyklė → Sujungimo operatorius → Kiti operatoriai (BONUS)

    DALIS 1: IntArray

    Tikslas: Suprasti Trijų Taisyklę (Rule of Three) izoliuotai, be kompozicijos, be vector.

    Kodėl IntArray? Paprasta klasė su int* ir dinamine atmintimi — idealus pavyzdys demonstruoti paviršinį (shallow) vs gilųjį (deep) kopijavimą. Primityvūs duomenys (int), paprasta logika, aiškiai matoma problema.

    Po IntArray pereisime prie labiau realaus use case — MyString su char*.

    Kopijos konstruktoriaus (copy constructor) sintaksė: 

    ClassName(const ClassName& other);

    Kada kviečiamas:

    • IntArray arr2 = arr1; — inicializacija
    • IntArray arr3(arr1); — inicializacija
    • IntArray arr4{arr1}; — universali inicializacija (C++11)
    • func(arr1); — perdavimas reikšme
    • return arr1; — grąžinimas reikšme
    • vector.push_back(arr1); — konteineriai

    Giliojo Kopijos (deep copy) algoritmas:

    1. Kopijuoti primityvius narius, pvz. size
    2. Sukurti NAUJĄ atmintį (new)
    3. Nukopijuoti DUOMENIS (ciklas arba memcpy)

    Svarbu!

    Kopijos priskyrimas (copy assignment) vs Kopijos konstruktorius (copy constructor):

    Atvejis Sintaksė Kviečia
    Inicializacija Type obj2 = obj1; Kopijos konstruktorius
    Inicializacija Type obj2(obj1); Kopijos konstruktorius
    Priskyrimas obj2 = obj1; Kopija priskyrimo operatorius

    Narių inicializavimo sąrašas (Member initializer list):

    Šiame žingsnyje pertvarkėme Kopijos konstruktorių, kad panaudoti narių inicializavimo sąrašą: Kodėl geriau?

    • Nariai inicializuojami, ne priskiriami (efektyviau)
    • Const nariai ir nuorodos privalo būti inicializuoti sąraše
    • Bazinės klasės konstruktoriai kviečiami sąraše
    • Geresnis stilius (Modern C++)

    Sintaksė: 

    ClassName::ClassName(params) 
    : member1(value1),
      member2(value2)
{
    // Kūnas
}

    Kopijos priskyrimo algoritmas:

    1. Savęs priskyrimo patikrinimas (savęs priskyrimo patikrinimas: if (this == &other))
    2. Išvalymas (cleanup: išvalyti seną atmintį)
    3. Gilusis kopijavimas (deep copy: nukopijuoti naują)
    4. return *this (grąžinti nuorodą)

    Kodėl return *this?

    • Įgalina grandinę (chaining): a = b = c;

    5 žingsnis: MyString - Sujungimas (operator+)

    Direktorija: U4/05/

    Motyvacija:

    Operatorių perkrovimas (operator overloading) — C++ galimybė apibrėžti, kaip standartiniai operatoriai (+, ==, [], <<) veikia su mūsų klasėmis. Tai daro kodą intuityvų ir natūralų:

    MyString s3 = s1 + s2;  // Vietoj: s3 = s1.concat(s2);

    Pradėsime nuo operator+ (sujungimas - concatenation).

    Reikalavimai:

    Pridėkite operator+ į MyString klasę.

    MyString.h (papildyti): 

    class MyString {
    // ... ankstesnis kodas ...
public:
    // ... ankstesnis kodas ...

    // NAUJAS: Sujungimas
    MyString operator+(const MyString& other) const;
};

    MyString.cpp (papildyti): 

    // Sujungimas
MyString MyString::operator+(const MyString& other) const {

    Testavimas:

    main.cpp: 

    #include <iostream>
#include "MyString.h"

int main() {
    std::cout << "=== MyString Sujungimas ===" << std::endl;

    MyString s1("Hello");
    MyString s2("C++");

    std::cout << "s1: \""; s1.print(); std::cout << "\"" << std::endl;
    std::cout << "s2: \""; s2.print(); std::cout << "\"" << std::endl;

    MyString s3 = s1 + s2;
    std::cout << "s3 = s1 + s2: \""; s3.print(); std::cout << "\"" << std::endl;

    // Grandinė (chaining)
    MyString s4 = s1 + s2 + MyString("!");
    std::cout << "s4 = s1 + s2 + \"!\": \""; s4.print(); std::cout << "\"" << std::endl;

    return 0;
}

    Svarbu!

    Operatoriaus perkrovimas (operator overloading) sintaksė: 

    ReturnType operator+(const Type& other) const;

    strcat vs strcpy: 

    strcpy(dest, src);   // Kopijuoja src → dest (pradžioj)
strcat(dest, src);   // Prijungia src prie dest (pabaigoj)
