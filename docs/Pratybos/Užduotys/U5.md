## Kontekstas: `has-a` vs `is-a`

Iki šiol dirbome su **has-a** (kompozicija): `Room HAS-A Window`, turėsime - `Shape HAS-A Point`

Dabar — **is-a** (paveldėjimas): `Circle IS-A Shape`, `Rectangle IS-A Shape`

**Svarbu!:** Fiziškai `Circle` objektas atmintyje **apima/įtraukia/turi** `Shape` dalį:

```
┌──────────────────┐
│  Shape: center   │  ← x, y
│  Shape: color    │
│  ──────────────  │
│  Circle: radius  │  ← papildomas laukas
└──────────────────┘
```
Techniškai:

`sizeof(Circle)` = `sizeof(Shape)` + `sizeof(double)`

### 1 žingsnis: Shape bazinė klasė

**Direktorija:** `U5/01/`

#### Motyvacija

Prieš paveldint — sukuriame **bazinę klasę** `Shape` ir pagalbinę `Point`. Tai apšilimas — pilnas pavyzdys, kurį naudosite visuose kituose žingsniuose.

Atkreipkite dėmesį: `Shape` turi `Point` (**has-a**) — tai kompozicija *viduje* bazinės klasės. Tuo pat metu `Circle` bus **is-a** `Shape`. Tai du **skirtingi** ryšiai tarp **skirtingų** klasių.

#### Kodas — PAVYZDYS

**Point.h:**
```cpp
#pragma once
#include <iostream>

struct Point {
    double x;
    double y;

    Point(double x = 0.0, double y = 0.0) : x(x), y(y) {}

    void print() const {
        std::cout << "(" << x << ", " << y << ")";
    }
};
```

**Shape.h:**
```cpp
#pragma once
#include "Point.h"
#include <string>

class Shape {
protected:
    Point center;
    std::string color;

public:
    Shape(double x, double y, const std::string& color = "black");
    ~Shape();

    Point getCenter() const;
    std::string getColor() const;
    void setColor(const std::string& color);

    double area() const;
    void printInfo() const;
};
```

**Shape.cpp:**
```cpp
#include "Shape.h"
#include <iostream>

Shape::Shape(double x, double y, const std::string& color)
    : center(x, y), color(color)
{
    std::cout << "[Shape CTOR] center=";
    center.print();
    std::cout << " color=" << color << "\n";
}

Shape::~Shape() {
    std::cout << "[Shape DTOR] color=" << color << "\n";
}

Point Shape::getCenter() const { return center; }
std::string Shape::getColor() const { return color; }
void Shape::setColor(const std::string& c) { color = c; }

double Shape::area() const { return 0.0; }

void Shape::printInfo() const {
    std::cout << "Shape [color=" << color << ", center=";
    center.print();
    std::cout << ", area=" << area() << "]\n";
}
```

**main.cpp:**
```cpp
#include <iostream>
#include "Shape.h"

int main() {
    std::cout << "=== 1 žingsnis: Shape bazinė klasė ===\n\n";

    Shape s1(0.0, 0.0);
    Shape s2(3.0, 4.0, "red");

    s1.printInfo();
    s2.printInfo();

    std::cout << "\nCentras: ";
    s2.getCenter().print();
    std::cout << "\n";

    std::cout << "\n--- Programa baigiasi ---\n";
    return 0;
}
```

#### Galima išvestis

```
=== 1 žingsnis: Shape bazinė klasė ===

[Shape CTOR] center=(0, 0) color=black
[Shape CTOR] center=(3, 4) color=red
Shape [color=black, center=(0, 0), area=0]
Shape [color=red, center=(3, 4), area=0]

Centras: (3, 4)

--- Programa baigiasi ---
[Shape DTOR] color=red
[Shape DTOR] color=black
```

#### Svarbu!

a) **`Point` kaip `struct` be `.cpp`** — `struct` skiriasi nuo `class` tik tuo, kad nariai pagal nutylėjimą yra `public`. `Point` yra paprastas duomenų konteineris — jokios logikos, jokių resursų. Visas kodas telpa į `.h` failą. Tai įprasta C++ praktika tokiems paprastiems tipams.

Kai `.h` failas savyje turi ne tik aprašus (_deklaracijas_), bet ir apibrėžimus (_definicijas_) — kaip čia `Point` — kai kurie programuotojai naudoja `.hpp` plėtinį. Abi konvencijos teisingos; `.hpp` aiškiau signalizuoja, kad faile yra ir implementacija. Su šablonais (_templates_) šis skirtumas taps svarbesnis.

b) **Default argumentai konstruktoriuje:**

```cpp
Point(double x = 0.0, double y = 0.0);
Shape(double x, double y, const std::string& color = "black");
```

`color = "black"` — jei kviesdami konstruktorių **nepateiksite** šio argumento, automatiškai bus naudojama reikšmė `"black"`. Pateiksite — bus naudojama jūsų:

```cpp
Shape s1(0, 0);           // color automatiškai "black"
Shape s2(0, 0, "red");    // color = "red"
```

Matėte tai jau U2 (`Student` konstruktoriuje) — čia tas pats principas.

c) **Inicializavimo sąrašas — kodėl geriau nei `this->`:**

```cpp
// Be sąrašo — priskyrimai kūne (blogesnis stilius):
Shape::Shape(double x, double y, const std::string& color) {
    this->color = color;   // color jau sukurtas su default, tada perrašomas
    center = Point(x, y);  // center jau sukurtas su default, tada perrašomas
}

// Su sąrašu — tiesiogiai inicializuojama (geresnis stilius):
Shape::Shape(double x, double y, const std::string& color)
    : center(x, y), color(color)
{ }
```

Sąraše `center(x, y)` — tai ne priskyrimas, o `Point` konstruktoriaus kvietimas. Kairėje — narys, dešinėje — parametras. Todėl `x(x)` sintaksė nekonfliktuoja — kompiliatorius supranta teisingai.

d) **Kompozicinis iškvietimas** — `s2.getCenter().print()`: `getCenter()` grąžina `Point` **reikšmę** (kopiją), todėl galime iš karto kviesti `.print()`. Tai tas pats `has-a` mechanizmas iš U3 — prieiga prie įkomponuoto poobjekčio per getterį. Grąžinama reikšmė, ne rodyklė — todėl `.`, ne `->`.

e) **Kodėl `area()` grąžina `0.0`?** — `Shape` nežino savo formos, ji bazinė, abstrakti. Konkretus plotas bus žinomas tik `Circle` ar `Rectangle` lygyje. Tai ne klaida — sąmoningas projektavimo sprendimas. Kaip tiksliai tai išspręsti — pamatysime vėliau.

f) **`protected` šiame žingsnyje dar "neatidirbo"** — 1 žingsnyje nėra jokių paveldėtojų, todėl `protected` ir `private` čia veiktų **vienodai**. Pakeitę `protected` į `private` — programa kompiliuosis ir veiks taip pat. `protected` prasmė atsiskleis tik kitame žingsnyje, kai `Circle` bandys pasiekti `center` ir `color`.

### 2 žingsnis: Circle paveldi Shape

**Direktorija:** `U5/02/`

#### Motyvacija

Pirmą kartą rašome `: public Shape`. Ką tai duoda praktiškai? `Circle` **automatiškai gauna** visus `Shape` metodus — `getColor()`, `getCenter()`, `setColor()`, `printInfo()` — nereikia jų rašyti iš naujo. Papildome tik tuo, kas `Circle` specifika: `radius`.

Konstruoti `Circle` objektą — reiškia konstruoti ir jo `Shape` dalį. Pagalvokite anatomiškai: kad gimtų vaikas, turi prieš tai "gimti" tėvas — t.y. `Shape` dalis turi būti sukurta **pirmiau** nei `Circle` dalis. Todėl `Circle` konstruktorius **privalo** pirmiausia iškviesti `Shape` konstruktorių:

```cpp
Circle::Circle(double x, double y, double r, const std::string& color)
    : Shape(x, y, color),   // ← pirma sukuriama Shape dalis
      radius(r)              // ← tada Circle dalis
{ ... }
```

Jei nenurodote — kompiliatorius ieško `Shape()` (be parametrų). Jo nėra — **kompiliavimo klaida**.

#### Svarbu!

a) **Kodėl `protected`, ne `private`?** — `Circle::printInfo()` naudoja `color` ir `center` tiesiogiai. Jei jie būtų `private` `Shape` klasėje — `Circle` negalėtų jų pasiekti net savo metoduose. `protected` = matoma klasėje ir jos paveldėtojuose, bet ne išoriniame kode:

```cpp

center.print();

// main() — ❌ nematoma išorėje (protected):
// c1.color = "red";
// c1.center.x = 5.0;
```

b) **Destruktorių tvarka** — naikinant `Circle`, pirma kviečiamas `Circle` destruktorius, tada `Shape` destruktorius. **Atvirkščia** konstruktorių tvarkai.

c) **`sizeof` rezultatai** — tikslūs skaičiai priklauso nuo sistemos. Svarbu pastebėti *skirtumą*: `Circle` objektas yra didesnis už `Shape`.

d) **`M_PI`** — kai kuriuose kompiliatoriuose reikia `#define _USE_MATH_DEFINES` prieš `#include <cmath>`. Alternatyva: `const double PI = 3.14159265358979;`

#### Svarbu!

a) **Trys perdavimo būdai — trys rezultatai:**

| Būdas | Slicing? | Interfeisas |

| `const Shape* s` — per rodyklę | ❌ ne | ribotas (kol kas) |

d) **5 žingsnis vs BONUS:**

- `Shape*` (5 žingsnis) — informacija **neprarandama**, bet interfeisas ribotas
- `Shape` reikšmė (BONUS) — informacija **prarandama** fiziškai
